# Path Layout

## Source of truth

```text
<repo>/        # git repo
~/.agents/    # local hub of symlinks into the repo
```

The two-level layout lets each person move the repo later and only update `~/.agents/*`.

## Repo paths

| Path | Purpose |
|---|---|
| `AGENTS.md` | Shared instructions for agents |
| `agents/` | Shared subagents |
| `instructions/` | Long-form instruction docs referenced by `AGENTS.md` |
| `skills/` | Versioned skills source tree |
| `.runtime/skills` | Generated flat skills runtime, ignored by Git |
| `claude/` | Claude settings, MCP template, statusline |
| `docs/` | Durable docs |
| `scripts/` | Sync and validation helpers |
| `.githooks/` | Optional local Git hooks |

## `~/.agents` symlinks

| Symlink | Target |
|---|---|
| `~/.agents/AGENTS.md` | `<repo>/AGENTS.md` |
| `~/.agents/agents` | `<repo>/agents` |
| `~/.agents/instructions` | `<repo>/instructions` |
| `~/.agents/skills` | `<repo>/.runtime/skills` |
| `~/.agents/claude` | `<repo>/claude` |

## Tool symlinks

| Tool | Symlink | Target |
|---|---|---|
| Claude | `~/.claude/settings.json` | `~/.agents/claude/settings.json` |
| Claude | `~/.claude/CLAUDE.md` | `~/.agents/AGENTS.md` |
| Claude | `~/.claude/agents` | `~/.agents/agents` |
| Claude | `~/.claude/skills` | `~/.agents/skills` |
| skills-cli | `~/.config/skills/skills.toml` | `<repo>/skills.toml` |

## Local-only files

| File | Why local |
|---|---|
| `~/.claude.json` | Claude Code live state; seeded once from `claude/claude.json`, never symlinked |
| `.runtime/` | Generated by `skills-cli` |

Pets are intentionally not part of this repo.
